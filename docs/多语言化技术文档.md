# 多语言化技术文档

## 概述

本文档详细描述了青枣工作室网站的多语言化实现方案，包括技术架构、文件结构、使用方法和扩展指南。

## 实现方案

### 1. 技术架构

本项目采用 React Context API 实现多语言支持，主要包含以下核心组件：

- **LanguageContext**: 语言上下文，管理全局语言状态
- **LanguageProvider**: 语言提供者组件，包裹整个应用
- **useLanguage**: 自定义 Hook，提供便捷的语言访问接口
- **LanguageSwitcher**: 语言切换组件，提供用户界面

### 2. 文件结构

```
src/
├── locales/                    # 语言资源文件目录
│   ├── zh-CN.json             # 中文语言资源
│   └── en-US.json             # 英文语言资源
├── contexts/                   # React Context 目录
│   └── LanguageContext.jsx    # 语言上下文和 Provider
├── hooks/                      # 自定义 Hooks 目录
│   └── useLanguage.jsx        # 语言 Hook
└── components/                 # 组件目录
    ├── LanguageSwitcher.jsx   # 语言切换组件
    ├── LanguageSwitcher.css   # 语言切换组件样式
    ├── Header.jsx             # 导航栏（已集成多语言）
    ├── Hero.jsx               # 首页横幅（已集成多语言）
    ├── Products.jsx           # 产品展示（已集成多语言）
    ├── About.jsx              # 关于我们（已集成多语言）
    ├── Contact.jsx            # 联系方式（已集成多语言）
    └── Footer.jsx             # 页脚（已集成多语言）
```

### 3. 语言资源文件格式

语言资源文件采用 JSON 格式，使用嵌套的键值对结构：

```json
{
  "header": {
    "logo": "青枣",
    "nav": {
      "products": "产品",
      "about": "关于",
      "contact": "联系"
    },
    "cta": "开始使用"
  },
  "hero": {
    "title": "创造实用工具",
    "subtitle": "青枣工作室专注于打造简洁高效的数字产品"
  }
}
```

**命名规范**：
- 使用点分隔符表示嵌套层级（如 `header.nav.products`）
- 键名使用驼峰命名法
- 保持中英文文件结构一致

### 4. 核心功能

#### 4.1 语言切换机制

用户点击语言切换按钮后：
1. 更新 React 状态（`language`）
2. 保存到 localStorage（`qingzao-language-preference`）
3. 更新 HTML `lang` 属性
4. 所有使用 `t()` 函数的组件自动重新渲染

#### 4.2 语言偏好记忆

- 首次访问：根据浏览器语言自动选择（中文浏览器 → 中文界面）
- 用户切换：保存到 localStorage，下次访问自动应用
- 存储键名：`qingzao-language-preference`

#### 4.3 翻译函数

```javascript
const { t } = useLanguage()

// 基础用法
t('header.logo')  // 返回: "青枣" 或 "Qingzao"

// 参数替换
t('footer.copyright', { year: 2024 })  // 返回: "© 2024 青枣. 保留所有权利."
```

## 使用方法

### 1. 在组件中使用多语言

```javascript
import React from 'react'
import { useLanguage } from '../hooks/useLanguage'

function MyComponent() {
  const { t } = useLanguage()

  return (
    <div>
      <h1>{t('hero.title')}</h1>
      <p>{t('hero.subtitle')}</p>
    </div>
  )
}
```

### 2. 添加新的翻译文本

**步骤 1**: 在语言资源文件中添加键值对

`zh-CN.json`:
```json
{
  "newSection": {
    "title": "新标题",
    "description": "新描述"
  }
}
```

`en-US.json`:
```json
{
  "newSection": {
    "title": "New Title",
    "description": "New Description"
  }
}
```

**步骤 2**: 在组件中使用

```javascript
<h2>{t('newSection.title')}</h2>
<p>{t('newSection.description')}</p>
```

### 3. 使用参数替换

当文本包含动态内容时，使用参数替换：

**语言文件**:
```json
{
  "stats": {
    "users": "{count} 用户使用"
  }
}
```

**组件使用**:
```javascript
t('stats.users', { count: 1000 })  // 返回: "1000 用户使用"
```

## 扩展指南

### 1. 添加新语言

**步骤 1**: 创建新的语言资源文件

创建 `src/locales/ja-JP.json`（日语示例）：

```json
{
  "header": {
    "logo": "青枣",
    "nav": {
      "products": "製品",
      "about": "について",
      "contact": "連絡"
    }
  }
}
```

**步骤 2**: 更新 LanguageContext

在 `src/contexts/LanguageContext.jsx` 中：

```javascript
// 导入新语言资源
import jaJP from '../locales/ja-JP.json'

// 添加到资源映射
const languageResources = {
  'zh-CN': zhCN,
  'en-US': enUS,
  'ja-JP': jaJP  // 新增
}

// 更新可用语言列表
const availableLanguages = [
  { code: 'zh-CN', name: '中文' },
  { code: 'en-US', name: 'English' },
  { code: 'ja-JP', name: '日本語' }  // 新增
]
```

**步骤 3**: 更新语言检测逻辑

```javascript
const getBrowserLanguage = () => {
  const browserLang = navigator.language || navigator.userLanguage

  if (browserLang && browserLang.startsWith('zh')) {
    return 'zh-CN'
  }
  if (browserLang && browserLang.startsWith('ja')) {
    return 'ja-JP'  // 新增
  }
  return 'en-US'
}
```

### 2. 语言文件命名规范

- 格式：`{语言代码}-{国家代码}.json`
- 示例：
  - `zh-CN.json` - 简体中文（中国）
  - `zh-TW.json` - 繁体中文（台湾）
  - `en-US.json` - 英语（美国）
  - `en-GB.json` - 英语（英国）
  - `ja-JP.json` - 日语（日本）

### 3. 翻译最佳实践

1. **保持一致性**：相同概念使用相同的翻译
2. **简洁明了**：避免过长的翻译文本
3. **文化适应**：考虑目标语言的文化习惯
4. **测试验证**：切换语言后检查所有页面

## 技术细节

### 1. React Context 实现

```javascript
// 创建上下文
export const LanguageContext = createContext(undefined)

// Provider 组件
export const LanguageProvider = ({ children }) => {
  const [language, setLanguageState] = useState(() => {
    const storedLang = getStoredLanguage()
    return storedLang || getBrowserLanguage()
  })

  const setLanguage = useCallback((lang) => {
    setLanguageState(lang)
    storeLanguage(lang)
    document.documentElement.lang = lang === 'zh-CN' ? 'zh-CN' : 'en'
  }, [])

  const t = useCallback((key, params) => {
    // 翻译逻辑...
  }, [language])

  return (
    <LanguageContext.Provider value={{ language, setLanguage, t, availableLanguages }}>
      {children}
    </LanguageContext.Provider>
  )
}
```

### 2. 翻译函数实现

```javascript
const t = useCallback((key, params) => {
  const keys = key.split('.')
  let value = languageResources[language]

  // 遍历键名获取对应的值
  for (const k of keys) {
    if (value && typeof value === 'object' && k in value) {
      value = value[k]
    } else {
      console.warn(`Translation key not found: ${key}`)
      return key
    }
  }

  // 参数替换
  if (params) {
    return value.replace(/\{(\w+)\}/g, (match, paramKey) => {
      return params[paramKey] !== undefined ? String(params[paramKey]) : match
    })
  }

  return value
}, [language])
```

### 3. 性能优化

- 使用 `useCallback` 缓存函数，避免不必要的重新渲染
- 语言资源文件在组件外部导入，避免重复加载
- localStorage 读写操作只在切换语言时执行

## 常见问题

### Q1: 如何处理缺失的翻译键？

A: `t()` 函数会返回键名本身，并在控制台输出警告信息。建议在开发阶段检查所有警告。

### Q2: 如何处理动态内容？

A: 使用参数替换功能：

```javascript
t('message.welcome', { name: userName })
```

### Q3: 如何处理复数形式？

A: 在语言文件中定义不同的键：

```json
{
  "items": {
    "one": "{count} 个项目",
    "many": "{count} 个项目"
  }
}
```

根据数量选择不同的键。

### Q4: 如何处理日期和数字格式？

A: 使用 JavaScript 内置的国际化 API：

```javascript
// 日期格式化
new Date().toLocaleDateString(language, { year: 'numeric', month: 'long', day: 'numeric' })

// 数字格式化
number.toLocaleString(language, { style: 'currency', currency: 'CNY' })
```
