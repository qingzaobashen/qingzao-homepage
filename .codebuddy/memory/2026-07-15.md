## GSC 未编入索引问题修复（qingzao-homepage）
- 现象：GSC 报告 12 个网页未编入索引，原因 = 自动重定向 / 404 / 重复网页(规范冲突)。
- 根因：prerender 产物是目录式 `dist/blog/<slug>/index.html`，仅「带斜杠」URL 命中真实内容；而 sitemap 与 canonical 都用「无斜杠」URL（`/blog/<slug>`）。Vercel 对目录 index.html 会把无斜杠 URL 308 重定向到带斜杠版本 → 被判为「自动重定向」；重定向后页面 canonical 仍是无斜杠，与 Google 实际收录的带斜杠 URL 不一致 → 「重复网页/规范冲突」；部分无斜杠 URL 无对应静态文件 → 404。
- 修复：`scripts/prerender.mjs` 的 `writeRoute` 改为生成扁平 `.html` 文件（`dist/blog/<slug>.html` 等），使「无斜杠」URL 直接命中真实静态文件（200、无重定向），与现有 sitemap/canonical（均为无斜杠）完全一致。
- 验证：`npm run build:full` 生成 22 个扁平 html；本地 `vite preview` 实测 `/blog/smart-home-system`、`/about`、`/blog`、`/privacy`、`/contact` 无斜杠 URL 均返回 200、无重定向，canonical=`https://qingzao.site/blog/smart-home-system`；Chrome DevTools 渲染正常、无 JS 报错。
- 关键约束：中英文文章共用同一 slug，静态文件只代表中文；英文由 SPA 客户端切换。sitemap/canonical 均保持无斜杠形式。
- 用户侧待办：重新部署后在 GSC 重新提交 sitemap，并对受影响 URL 用「网址检查」请求编入索引（GSC 重新处理需数日）。

## URL 级多语言（方案 A）：让英文有独立可索引 URL（qingzao-homepage）
- 背景：用户指出「只预渲染中文，英文对 Google 不可见」。根因不是预渲染本身，而是原架构中英文共用同一 slug、靠 localStorage/浏览器语言切换，没有独立 URL。
- 决策：实施方案 A——URL 路径前缀区分语言。中文默认无前缀（`/blog/<slug>`），英文加 `/en`（`/en/blog/<slug>`、`/en/about` 等）。
- 改动点：
  1. `src/contexts/LanguageContext.jsx`：语言真源改为 URL（`/en` 前缀 → en-US，否则 zh-CN）；`setLanguage` 改为「在同内容的中/英路径间 `navigate`」；新增 `localePath(path)` 给路径加 /en 前缀；移除对浏览器语言的默认依赖（URL 为规范真源）。
  2. `src/App.jsx`：新增 `/en` 镜像路由（复用同一页面组件）；首页 SEO 标题/描述/canonical 随语言动态化。
  3. `src/components/SEO.jsx`：新增 `alternates`（hreflang）prop；`siteName`/`og:locale` 随语言（英文 "Qingzao Studio"/en_US）。hreflang 由 Helmet 客户端输出。
  4. 博客/静态页 + Header/Footer/Hero/ContentHighlights/CookieConsent 内部链接全部改用 `localePath`，保证中英文站内跳转不错语言。
  5. `scripts/prerender.mjs`：为每篇博客与每个静态路由各生成中/英两份扁平静态 HTML（`dist/blog/<slug>.html` 与 `dist/en/blog/<slug>.html` 等）；英文读 `-en.md`；注入 `canonical`/`og`/`html lang`/`og:locale`。**hreflang 不在预渲染注入**（改由 Helmet 客户端输出），避免与 Helmet 重复生成两份 `<link rel=alternate>`。
  6. `public/sitemap.xml`：用一次性脚本为每个 `<url>` 注入 `xhtml:link` hreflang（zh-CN/en-US/x-default），并加 `xmlns:xhtml` 命名空间；x-default 指向中文 URL。
- 验证：`npm run build:full` 成功；Chrome DevTools 实测：
  - 直接加载 `/en/blog` → 英文列表、全部 `/en` 链接、head canonical=`https://qingzao.site/en/blog`、hreflang 三组、`htmlLang=en`、标题 "Blog - Qingzao Studio"。
  - 直接加载 `/blog` 与 `/en/blog/house-acceptance-inspection` → canonical 各正确（`/blog` 与 `/en/blog/...`）、hreflang 正确、正文语言正确。
  - 语言切换器点击「中文」从 `/en/blog` 导航到 `/blog`（URL 级切换生效）。
  - 顺带修复：`en-US.json` 页脚 Twitter 链接键名大小写不一致导致英文页显示原始 key 的 bug（改为 "Twitter" 与 zh-CN/代码一致）。
- 已知小瑕疵：SPA 客户端在 `/en/blog`→`/blog` 这种会话内导航时，Helmet 未能覆盖预渲染 HTML 继承来的 canonical（会短暂残留上一页的 canonical）；但 Google 直接抓取 URL（冷加载）时 canonical 正确，不影响索引。属预渲染+Helmet 既有交互，未专门处理。
