## GSC 未编入索引问题修复（qingzao-homepage）
- 现象：GSC 报告 12 个网页未编入索引，原因 = 自动重定向 / 404 / 重复网页(规范冲突)。
- 根因：prerender 产物是目录式 `dist/blog/<slug>/index.html`，仅「带斜杠」URL 命中真实内容；而 sitemap 与 canonical 都用「无斜杠」URL（`/blog/<slug>`）。Vercel 对目录 index.html 会把无斜杠 URL 308 重定向到带斜杠版本 → 被判为「自动重定向」；重定向后页面 canonical 仍是无斜杠，与 Google 实际收录的带斜杠 URL 不一致 → 「重复网页/规范冲突」；部分无斜杠 URL 无对应静态文件 → 404。
- 修复：`scripts/prerender.mjs` 的 `writeRoute` 改为生成扁平 `.html` 文件（`dist/blog/<slug>.html` 等），使「无斜杠」URL 直接命中真实静态文件（200、无重定向），与现有 sitemap/canonical（均为无斜杠）完全一致。
- 验证：`npm run build:full` 生成 22 个扁平 html；本地 `vite preview` 实测 `/blog/smart-home-system`、`/about`、`/blog`、`/privacy`、`/contact` 无斜杠 URL 均返回 200、无重定向，canonical=`https://qingzao.site/blog/smart-home-system`；Chrome DevTools 渲染正常、无 JS 报错。
- 关键约束：中英文文章共用同一 slug，静态文件只代表中文；英文由 SPA 客户端切换。sitemap/canonical 均保持无斜杠形式。
- 用户侧待办：重新部署后在 GSC 重新提交 sitemap，并对受影响 URL 用「网址检查」请求编入索引（GSC 重新处理需数日）。
