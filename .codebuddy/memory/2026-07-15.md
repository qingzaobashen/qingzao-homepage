## P2 首页文章链接语言前缀 bug
- 现象：英文版首页 `/en` 点文章卡片跳到中文 `/blog/:slug`（语言不一致）。
- 根因：`src/components/ContentHighlights.jsx` 第 45 行卡片链接硬编码 `/blog/${slug}`，未用 `localePath()`（底部 CTA 按钮已用，故仅卡片出错）。
- 修复：改为 `to={localePath(`/blog/${post.slug}`)}`，与 `BlogListPage.jsx` 一致。
- 验证：dev server 22333，英文首页 `/en` 三卡片 href 现均为 `/en/blog/...`，正常。
