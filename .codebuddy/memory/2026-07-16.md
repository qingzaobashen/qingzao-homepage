# 2026-07-16

## 博客评论系统实现（qingzao-homepage）
为博客详情页实现完整评论功能，集成进 `BlogPostPage.jsx`（返回链接之后、Footer 之前）。

新增文件：
- `src/api/comments.js`：模块化后端接口层。定义与 Supabase 表 `comments`（id/post_slug/parent_id/author_name/content/created_at/updated_at）一一对应的数据模型；`SupabaseCommentRepository` 预留 `select().eq().order()` / `insert().select().single()` / `update().eq().select().single()` 方法（动态 `import('@supabase/supabase-js')` 用变量名 + `@vite-ignore`，构建期不强制解析该依赖）；未配置 `VITE_SUPABASE_URL`/`VITE_SUPABASE_ANON_KEY` 时自动降级到 `LocalCommentRepository`（localStorage 持久化）。对外暴露 `fetchComments / createComment / updateComment`。
- `src/components/comments/`：`CommentSection.jsx`（容器，加载/错误/提交状态 + 树组装）、`CommentItem.jsx`（递归嵌套，至少二级）、`CommentForm.jsx`（昵称/正文 + 字数限制与空值校验）、`comments.css`（响应式）。
- `zh-CN.json` / `en-US.json` 增加 `comments` 翻译键。

测试发现并修复的 Bug：紧凑回复表单原本不渲染昵称输入框，但校验仍要求昵称 → 回复永远提交失败。已改为回复表单也渲染昵称输入框（compact 仅减少内边距/行高）。Chrome DevTools 实测：顶级评论 parent=null，二级回复 parent_id 正确指向父评论 id，并经 `.comment-item__children` 嵌套渲染。

注：项目 rules 要求网页改动用 Chrome DevTools MCP 验证；本环境用 `vite preview --port 4173` 预览（导航常因外部资源 ERR_NAME_NOT_RESOLVED 超时，但 SPA 正常渲染）。
