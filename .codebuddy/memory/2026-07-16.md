## 博客图片展示功能（Markdown 图片语法）
在 `src/data/posts/markdownToHtml.js` 中新增标准 Markdown 图片解析 `![alt](url "title")`，运行时与预渲染共用同一逻辑：
- 新增 `renderImage(alt,url,title,block)`、`escapeHtml/escapeAttr` 辅助函数与常量 `IMG_ONERROR`。
- 行内图片在 `renderInline` 中解析，**必须早于链接规则**（否则 `[alt](url)` 会被链接规则误吞），产出 `<span class="content-image-inline">`。
- 整行仅为一张图片时在主循环解析为块级 `<figure class="content-figure">`，含 `<figcaption>`（alt 非空时）。
- 图片属性：`loading=lazy`、`decoding=async`、`alt`（SEO/无障碍）。
- 加载失败降级：内联 `onerror` 隐藏损坏图片并显示同级 `.content-image-fallback` 占位符（展示 alt 文本，空 alt 时展示「图片加载失败 / Image failed to load」）。因正文经 dangerouslySetInnerHTML/预渲染注入且项目无 CSP，采用内联 onerror。
- 样式在 `src/pages/BlogPostPage.css` 新增：`.content-figure/.content-image/.content-image-inline/.content-figcaption/.content-image-fallback`，图片 `max-width:100%;height:auto` 自适应等比缩放，并在 768px 媒体查询做移动端收紧。
- 已用临时 node 脚本验证解析（行内/块级/空 alt/链接兼容均正确），`npm run build:full` 通过，lint 0 错误。
